# Групповой проект 2

## Описание

Этот проект включает в себя несколько этапов для исследования корреляции стоимости жилья с его расположением относительно инфрасструктуры города Москва. Для этого мы используем **парсинг** данных с сайта **Cian** и **интеграцию с API OpenStreetMap** для получения точных координат объектов. Сравнение стоимости недвижимости с ближайшими удобствами позволяет сделать более точную оценку для каждого объекта недвижимости.

Процесс включает:
1. Сбор данных с сайта недвижимости Cian.
2. Получение координат общественных объектов через API OpenStreetMap.
3. Сравнение данных и анализ влияния близости к объектам инфраструктуры на стоимость жилья.

## Процесс работы

### 1. **Парсинг данных с сайта Cian**

Процесс начинается с **парсинга данных** с сайта недвижимости **Cian**. Так как на сайте пролистывать дальше 54 страницы, мы прошлись по всем страницам каждого районам и собрали все ссылки на объявления в отдельный файл. Проходимся по каждой ссылке в нашем датасете  для извлечения информации о недвижимости, включая:

- Адрес
- Количество комнат
- Стоимость квадратного метра 
- Площадь
- Этажность
- Тип недвижимости (квартира, дом и т.д.)
- Расстояние до метро (время)

Оба наших парсинга были выполнены при помощи библиотеки **Selenium**, в парсинге сайта  **Cian** для извлечения ссылок на объявления, во второй раз, мы прошлись по всем ссылкам и собирали всю необходимую информацию. Мы собираем эти данные в таблицу и готовим их для дальнейшего анализа.

#### Пример:

1. Получаем данные о квартире: цена, площадь, этаж.
2. Собираем данные для нескольких объектов в заданном районе..

### 2. **Получение координат объектов через OpenStreetMap API**

Далее мы используем **OpenStreetMap API** для получения координат различных объектов инфраструктуры (школы, детские сады, поликлиники, парки), расположенных рядом с найденными объектами недвижимости.

Процесс включает в себя:

- Отправку запроса к API OpenStreetMap для поиска объектов по ключевым словам (например, "school", "kindergarten", "clinic", "park").
- Получение координат (долгота и широта) найденных объектов.
- Привязку координат к конкретному объекту недвижимости для дальнейшего анализа.

### 3. **Сравнение и анализ данных**

После того как мы получили все необходимые данные, начинается процесс **сравнения**:

- Для каждой квартиры или дома, найденного на Cian, мы вычисляем расстояние до ближайших объектов инфраструктуры с помощью библиотеки **geopy**.
- На основе полученных расстояний, мы оцениваем стоимость недвижимости, учитывая близость к объектам (например, квартиры ближе к школам могут иметь более высокую цену).
- Мы создаем таблицы и визуализации для удобного анализа.

Этот процесс позволяет не только собрать данные, но и сделать выводы о влиянии расположения недвижимости на ее стоимость.
## Этап EDA (Exploratory Data Analysis)

**На этапе анализа данных мы провели несколько ключевых шагов для предварительного исследования и подготовки данных:**

 **Очистка данных:** Убрали ненужные данные,Nanы,исключили информацию о валюте и единицах измерения (например, от квадратных метров).

 **Анализ данных:** Построили шкалу цен в миллионах,рассмотрели распределение данных, чтобы понять, как варьируются цены на жильё.

 **Вычисление цены за квадратный метр:** Выделили цену за 1 м², что позволяет лучше понять стоимость недвижимости.

 **Анализ распределения цен:** Нашли распределение цен за квадратный метр, что помогает понять, какие районы или типы недвижимости могут быть дешевле или дороже.

 **Анализ метро и станций:** Проанализировали ближайшие станции метро и выявили 10 самых дорогих объектов рядом с ними.

 **Исследование влияния типа дома и расстояния до метро:** Рассмотрели, как тип дома (квартира, дом и т.д.) и расстояние до ближайшей станции метро влияют на цену недвижимости.

 **Анализ медианы и интерквартильного диапазона:** Исследовали медиану и интерквартильный диапазон цен и годов постройки для более точной оценки рынка недвижимости.
 **Цена за минуту до метро:** Рассчитали цену за минуту до ближайшего метро и сравнили её с ценой за квадратный метр, чтобы определить взаимосвязь между ними.

 **Матрица корреляций:** Создали матрицу корреляций для выявления взаимосвязей между различными переменными (например, тип дома, цена, расстояние до метро) и сделали выводы по полученным данным.


## Функции используемые с нашем проекте

- **Парсинг данных с Cian:** Скачиваем и сохраняем информацию о недвижимости, включая цену, площадь и другие характеристики.
- **Получение координат:** Используем OpenStreetMap API для получения точных координат объектов инфраструктуры.
- **Использоване GeoAPIPython** После получения координат, для большего удобства в работе, ипользуем геокодинг.
- **Сравнение данных:** Рассчитываем расстояние между объектами недвижимости и ближайшими объектами инфраструктуры.
- **Оценка стоимости:** Сравниваем стоимость недвижимости с учетом близости к важным объектам инфраструктуры и создаем отчеты.
